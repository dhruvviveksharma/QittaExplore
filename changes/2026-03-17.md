## 1. Barnacle Integration — Configurable API Base + Startup Script

### Problem
The Flask backend requires `qiita_db.sql_connection.TRN`, which only works inside the Qiita conda environment on barnacle (it connects to `qiita-db-rc` PostgreSQL). The frontend had the API URL hardcoded to `http://localhost:5001/api`, and there was no startup script to launch Flask on barnacle with the correct environment and config.

### Solution
Made the API base URL configurable via an HTML meta tag (no JS change needed to switch environments), added a barnacle startup script that hard-codes the correct absolute config path, and excluded `qiita_config.cfg` from git (it contains the DB password).

### Dev Workflow
```bash
# On barnacle (Terminal 1):
bash ezredbiom/Experiment/start_barnacle.sh

# On local machine (Terminal 2 — SSH tunnel):
ssh -N -L 5001:localhost:5001 d4sharma@barnacle2

# On local machine (Terminal 3 — serve frontend):
cd ezredbiom/Experiment/frontend
python -m http.server 3000
# Open: http://localhost:3000
```

### Files Changed

#### `ezredbiom/Experiment/frontend/index.html`
- Added `<meta name="api-base" content="http://localhost:5001/api">` inside `<head>`
- This tag is the single place to change the API URL when switching between dev tunnel and direct barnacle URL — no JS edits required

#### `ezredbiom/Experiment/frontend/app.js`
- Line 3: replaced hardcoded `const API = 'http://localhost:5001/api'` with:
  ```js
  const API = document.querySelector('meta[name="api-base"]')?.content
            || 'http://localhost:5001/api';
  ```

#### `ezredbiom/Experiment/start_barnacle.sh` *(new file)*
- Activates `qiita-web` conda env (which has `qiita_db` installed)
- Hard-codes `QIITA_CONFIG_FP=/home/d4sharma/qiita-web/qiita_config.cfg` — absolute path prevents breakage when an inherited relative path is in the shell environment
- Sets `QIITA_EXPERIMENT_DB_PATH=$HOME/.qiita-experiment/projects.db` for persistent SQLite storage
- `cd`s into `backend/` and runs `python run.py`

#### `.gitignore`
- Added `qiita_config.cfg` — this file contains the PostgreSQL password for `qiita-db-rc` and must never be committed

## 2. Sample Preview Card — Click a Sample Row to Inspect Metadata

### Problem
The study modal's Samples table showed only 4 columns (Sample ID, Anonymized Name, Env Package, Collection Date). Each sample in Qiita has a rich `sample_values` JSONB column in `qiita.sample_{study_id}` with dozens of metadata fields (body site, collection method, host subject ID, DNA extraction details, etc.) that were not accessible from the UI.

### Solution
Clicking any row in the Samples table now opens a sticky preview card to the right of the table, showing all non-empty key→value pairs from the sample's JSONB metadata sorted alphabetically. Clicking the same row again or pressing ✕ dismisses the card. The card resets when the modal is closed or a different study is opened.

### Architecture
```
User clicks sample row
  → fetchSamplePreview(studyId, sampleId)
      → GET /api/studies/{id}/samples/{sample_id}
          → SELECT sample_values FROM qiita.sample_{study_id}
            WHERE sample_id = %s AND sample_id <> 'qiita_sample_column_names'
          → return {sample_id, fields: {key: value, ...}}
  → samplePreview state set → preview card renders
```

### Files Changed

#### `ezredbiom/Experiment/backend/run.py`
- **New endpoint: `GET /api/studies/<study_id>/samples/<path:sample_id>`**
  - Queries `qiita.sample_{study_id}` for `sample_values` JSONB
  - Filters out the sentinel row `qiita_sample_column_names`
  - Strips internal key `qiita_study_id` from the returned dict
  - Returns `{sample_id, fields: {…}}` or 404 if not found

#### `ezredbiom/Experiment/frontend/app.js`
- **New state variables** (near other modal state):
  - `samplePreview` — `null` or `{sample_id, fields}` object from API
  - `samplePreviewLoading` — boolean, shows "Loading…" while fetching
- **New function `fetchSamplePreview(studyId, sampleId)`**:
  - Toggles off (clears state) if the same row is clicked again
  - Sets `samplePreviewLoading` true, fetches `/studies/{id}/samples/{sampleId}`, sets `samplePreview`
  - Called with `encodeURIComponent(sampleId)` to handle dots in IDs (e.g. `77.TS100.2`)
- **Reset on modal lifecycle**: `samplePreview` cleared to `null` in `closeModal()` and at the start of `openStudyModal()` to prevent stale state on re-open
- **Clickable sample rows**: each `<tr>` now has `onClick`, `cursor:pointer` style, and a highlighted background (`var(--accent-bg)`) when selected
- **Two-column samples section layout**: outer `<div>` is a flex row
  - Left column (55% max-width): existing `samples-table-wrap` with the prep-table
  - Right column (flex:1): loading indicator or preview card
- **Preview card JSX**:
  - Header: sample ID + ✕ close button (`sample-preview-close` class)
  - Body: `Object.entries(fields)` filtered for non-empty values, sorted alphabetically, rendered as `sample-preview-row` divs with key/value spans

#### `ezredbiom/Experiment/frontend/style.css`
- **`.sample-preview-card`** — flex:1, sticky positioning, max-height 280px, scrollable, rounded border
- **`.sample-preview-header`** — flex row (space-between), sticky top, font-weight 600, border-bottom
- **`.sample-preview-close`** — borderless button, hover changes color to `var(--text-1)`
- **`.sample-preview-body`** — minimal padding wrapper
- **`.sample-preview-row`** — flex row, 0.78rem font, border-bottom (none on last child)
- **`.sample-preview-key`** — `var(--text-3)`, flex `0 0 44%`, word-break
- **`.sample-preview-val`** — `var(--text-2)`, flex:1, word-break

## 3. Git Branch Reconciliation

### Problem
Local branch `Local-Dev` and `origin/Local-Dev` had diverged (1 commit each): the remote had a `start_barnacle.sh` addition; local had a session commit. `git pull` failed with "Need to specify how to reconcile divergent branches."

### Solution
Committed the local working directory changes, then ran `git pull --rebase origin Local-Dev`. Both commits were replayed cleanly onto a single linear history — no merge commit.

```
Before:                         After:
  origin: A → B → R             origin: A → B → R
  local:  A → B → L                      ↓
  (diverged at B)               local:  A → B → R → L (rebased)
```
